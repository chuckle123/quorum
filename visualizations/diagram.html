1 TRIGGER
An external event kicks off the process. A ticket arrives, a PR lands, a metric spikes.
Customer writes in:
"Checkout fails on mobile when using Apple Pay"
Coding agent finishes a PR that introduces a new error boundary pattern in React components.
Churn dashboard shows 23% increase in cancellations over the last 30 days. Growth team needs a response.

An agent with the right context:
• Observes the trigger
• Searches existing memory
• Creates a new thread
• Tags relevant agents

4 comments · Queue empty · TERMINATE → VOTE

5 VOTING PHASE
Each thread_agent votes.
No tools. Full thread as context.
Returns: { value: +1|-1, reasoning }

Rules:
• Min 2 votes cast
• 51%+ of ALL thread_agents approve

+1 backend "Confirmed root cause"
+1 design "UX fix needed too"
+1 product "P1, ship it"
4/4 approve
✓ APPROVED
+1 coding "This is the pattern I proposed"
+1 frontend "Clean, matches standards"
+1 architect "Good pattern, with constraint"
3/3 approve
✓ APPROVED
+1 growth "Worth trying, even at lower %"
−1 brand "Devalues premium positioning"
−1 finance "Unit economics don't work"
−1 cx "Addresses symptom, not cause"
1/4 approve
✗ REJECTED
